# 提交信息模板

## 章节总结

### 基本模式

```
docs(书名目录): 新增第 N 章总结 - 章节标题

核心：一句话核心观点。
- 要点 1
- 要点 2
- 要点 3
```

### 真实示例（来自《持续买入》仓库）

```
docs(持续买入): 新增第 13 章总结 - 为什么不应该等待逢低买入

核心：即使上帝也打不过定期定额投资法（DCA）。
- 思想实验：1920-1980 任意 40 年区间，拥有完美择时的"逢低买入"仍有 70%+ 时间输给 DCA
- 原因：市场暴跌极少，大多数时间在新高附近
- 经典反例 1975-2014：熊底已过，下一个历史高点要等 10 年
- 致命数据：错过底部 2 个月胜率从 30% 暴跌到 3%
- 持续买入数据：1926 起任意月开始 10 年持续买美股，98% 跑赢持币
```

## README 更新

### 当章节更新时同步

```
docs: 更新 README 索引，补充第 N 章链接
```

### 或者合并进章节总结

更推荐做法：在同一次 commit 里包含新的章节文件 + README 更新，commit message 末尾加一句"README 同步更新"。

## 延伸阅读（独立文件）

```
docs: 新增 <书名目录> 延伸阅读 - <主题>

独立文件，便于日后单独回看。核心：
- 要点 1
- 要点 2
- 要点 3
```

### 真实示例

```
docs: 新增红利低波 10 年最坏情境沙盘

独立文件便于日后单独回看。内容包括：
- 一句话总结
- 起点设定与历史参照系
- 10 年分阶段路径推演
- 与其他资产最坏情景对比
- 四问法回测承受力
- 通用建仓决策流程
```

## 规则总结

1. 中文 commit message
2. 首行格式：`docs(目录): 动作 + 对象`
3. 空一行后写"核心：一句话"作为内容导引
4. 接一个要点列表（3–7 条），每条不要超过一行
5. 不要写"本次提交"、"本次新增"这种多余词
6. 涉及具体数字/事实的直接引用原文数字（"76% 时间跑输"比"多数时间跑输"更有价值）
